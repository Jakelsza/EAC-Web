How to use the companion

          
Everything you need for a clean Abyss run, in order.


          

            
1 · One-time setup

            
On first launch the wizard asks for your EVE Gamelogs folder — usually
              Documents\EVE\logs\Gamelogs. This is how the companion sees NPCs:
              it only ever reads the log files EVE writes to disk. No memory reading, no game hooks,
              nothing that touches the client. You can change the folder any time under Systems.

            
Folder access and the floating overlay need a Chromium browser (Chrome / Edge / Opera GX)
              opened directly on this file.

          


          

            
2 · The run timer

            
Pick your Tier and Weather, enter your cargo's estimated value in Before,
              and hit ▶ Start as you take the filament gate. The ring counts down the 20-minute Abyss
              limit and turns red under five minutes. Audio cues fire at 5:00, 1:00 and 0:00 (toggle in Systems).

            
Tap R1 ✓ / R2 ✓ / R3 ✓ as you clear each room. Each split shows the time spent
              in that room only — not cumulative — so you can see exactly where time is lost.
              Marking R3 stops the clock automatically and waits for you to enter the loot total.

            
After submitting a run the After value carries forward as the next run's Before
              automatically — no re-typing between runs.

            
Tier and Weather lock while a run is in progress and unlock again on reset.

          


          

            
3 · NPC detection

            
During a run the companion watches the log and lists every hostile it sees, colour-coded by
              the trouble they bring. Click any NPC chip to open a full info panel showing threat,
              EWAR type, kill priority, and notes — without shifting the layout mid-run.

            
    
              
  • Scram — warp scrambler, you're held
    • 
              
  • Neut — energy neutraliser, drains your cap
    • 
              
  • Web — stasis web, slows you
    • 
              
  • Damp/Dis — sensor damp / tracking disruption
    • 
              
  • Logi — remote reps; kill these first
    • 
              
  • Paint — target painter
    • 
              
  • Gold border — room boss · the gold chip is the latest spawn
    • 
            

            
The chip list resets each room so only the current room's threats show. If your fit
              has no drones, switch off I use drones in Systems to hide the drone-repair flag.

          


          

            
4 · The overlay

            
Overlay ▶ opens a floating always-on-top window (Picture-in-Picture) you can position
              over EVE in windowed-fullscreen. It carries the countdown timer, tier/weather selectors,
              room buttons, and loot fields — everything you need without alt-tabbing. It resizes freely
              and all content scales to fit.

            
By default the overlay shows only the controls. Turn on Show NPC names in overlay
              under Systems to add a lightweight NPC panel: name and a kill-priority colour dot per hostile,
              with no wall of text. Veterans keep it off; newer pilots turn it on.

            
The full clickable NPC info panel lives in the main app only, not the overlay.

          


          

            
5 · Loot, history & stats

            
After the run, enter your cargo's new value in After and hit ✓ Submit & End
              (or ■ End any time to log a partial run without loot). The difference is your loot delta.

            
Loss runs — where the delta is negative — show the delta in red with a red left border
              on the history row. Records are shown as small badges on each run:

            
    
              
  • T — fastest run for this tier + weather
    • 
              
  • $ — best loot for this tier + weather
    • 
              
  • ★ — best ISK/hr for this tier + weather
    • 
            

            
Stats breaks down ISK/hour from your own run history and shows
              which tier + weather combination pays best. Switch between All-time and per-session scope using
              the buttons at the top of the Stats tab.

          


          

            
6 · History views

            
The History tab has two views: Runs shows all standalone runs (logged outside a session)
              in reverse chronological order. Sessions shows past and active sessions as collapsible
              rows — expand any session to see all its runs, with Edit and Delete buttons on each.

            
You can edit any run to fix the Before/After loot values, tier, or weather. You can
              delete individual runs or entire sessions from the expanded session view.

          


          

            
7 · Sessions

            
Planning a block of runs? In Stats, name a session and set a target run count. The
              companion groups all runs logged during that session, shows your progress toward the target,
              and saves the full session when you're done. Past sessions are preserved — compare a Tuesday
              grind to a Wednesday one, or scope your stats to any single session using the session picker.

            
Tier and weather lock at the start of a session run and unlock when you reset or submit.

          


          

            
8 · NPC Library

            
The NPC Library tab is a searchable, faction-filterable reference for all 126 NPCs in the
              bundled dataset. Click any entry to see full stats: damage type, resistances, EWAR, kill
              priority, and notes. The same dataset powers the auto-detection — no separate import needed.

            
If you have a custom dataset, import it under Systems → Import dataset (same
              npc_dataset.json schema). Restore the bundled set any time with
              Restore bundled dataset.

          

        

      


      

        

          
EVE Abyss Companion

          
v2.3.0 · Web Edition — “Built by capsuleers, for capsuleers.”

          
A free companion for Abyssal Deadspace runs. It reads your EVE Online game log files from disk to
            detect NPCs — no memory reading, no game hooks, no injection, no EVE API calls. Everything runs and
            stays on your machine: settings, history and your NPC dataset never leave this file.

          
Requirements: a Chromium browser (Chrome, Edge, or Opera GX) opened directly on this file.
            Folder access and the always-on-top overlay use the File System Access and Document Picture-in-Picture
            APIs, which Firefox does not currently ship.
